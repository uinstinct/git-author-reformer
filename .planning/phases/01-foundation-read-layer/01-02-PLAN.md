<feature>
  <name>Pre-flight safety gates: stash and worktree detection</name>
  <files>src/git/preflight.rs, tests/preflight_test.rs</files>
  <behavior>
    Two pure functions, each takes `&Repository`, returns `Result<(), AppError>`.

    `check_stash` test cases:
    1. Clean repo (no refs/stash) → `Ok(())`
    2. Repo with `refs/stash` pointing at HEAD → `Err(AppError::StashDetected)`

    `check_worktrees` test cases:
    1. Repo with no linked worktrees → `Ok(())`
    2. Repo with one linked worktree → `Err(AppError::WorktreesDetected(name))` where name contains the linked worktree's name
  </behavior>
  <implementation>
    Per 01-RESEARCH.md Patterns 3 and 4:
    - `check_stash`: `if repo.find_reference("refs/stash").is_ok() { return Err(AppError::StashDetected); } Ok(())`
    - `check_worktrees`: `let worktrees = repo.worktrees()?; if !worktrees.is_empty() { let names: Vec<&str> = worktrees.iter().flatten().collect(); return Err(AppError::WorktreesDetected(names.join(", "))); } Ok(())`
    - `worktrees.is_empty()` reflects libgit2 behavior: the main worktree is NOT included in `Repository::worktrees()`, so an empty result truly means no linked worktrees (Pitfall 4 in RESEARCH.md).
  </implementation>
</feature>
