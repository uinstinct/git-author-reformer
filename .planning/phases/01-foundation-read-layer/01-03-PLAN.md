<feature>
  <name>Branch-reachable enumeration of authors and co-authors</name>
  <files>src/git/reader.rs, tests/reader_test.rs</files>
  <behavior>
    `enumerate_authors` cases:
    1. Empty repo → returns empty Vec (no panic, no error)
    2. Repo with Alice (1 commit) and Bob (2 commits) → returns `[Bob (2), Alice (1)]` — sorted desc
    3. Same name with different emails counted separately → two entries
    4. Non-UTF-8 author field → does not panic (treated as empty string via `unwrap_or("")`)

    `enumerate_coauthors` cases:
    1. Three commits with `Co-authored-by:`, `CO-AUTHORED-BY:`, `co-authored-by:` (same Name+Email) → dedupes to a single entry with count 3
    2. Repo with no co-author trailers → returns empty Vec
    3. Malformed `Co-authored-by: just text no angle brackets` → skipped, no panic
    4. Two distinct co-authors → two entries, sorted by count desc
  </behavior>
  <implementation>
    Per 01-RESEARCH.md:
    - Pattern 2: private `build_revwalk(repo)` helper using `push_glob("refs/heads/*")` and `Sort::TOPOLOGICAL | Sort::REVERSE`
    - Pattern 5: `enumerate_authors` walks all commits, uses `HashMap<(String, String), usize>` to count, sorts descending. Use `unwrap_or("")` on `author.name()` and `author.email()` (Pitfall 2).
    - Pattern 6: `enumerate_coauthors` walks all commits, scans message lines, uses helper `strip_coauthor_prefix` (case-insensitive ASCII prefix match) and `parse_coauthor_value` (rfind('<')/'>' to extract Name+email). Both helpers are private fn in reader.rs.
    - The two private helpers may be unit-tested in-module via `#[cfg(test)] mod tests` at the bottom of `reader.rs` if desired, but the integration tests in `tests/reader_test.rs` are the contract.
  </implementation>

    - Private `fn build_revwalk(repo: &git2::Repository) -> Result<git2::Revwalk<'_>, git2::Error>` calling `revwalk.push_glob("refs/heads/*")?` and `revwalk.set_sorting(Sort::TOPOLOGICAL | Sort::REVERSE)?`. CRITICAL: use `refs/heads/*` ONLY — never `refs/*` (would include synthetic stash commits and inflate counts) and never `refs/tags/*` (annotated tags point at tag objects, not commits — `push_glob` does not auto-peel and will fail at runtime per Pitfall 1).
    - `enumerate_authors`: build revwalk, iterate, `let commit = repo.find_commit(oid?)?`, extract `commit.author().name().unwrap_or("").to_string()` and `commit.author().email().unwrap_or("").to_string()` (Pitfall 2 — never `.unwrap()`), accumulate counts in `HashMap<(String, String), usize>`, convert to `Vec<AuthorIdentity>`, sort by `commit_count` descending via `sort_by(|a, b| b.commit_count.cmp(&a.commit_count))`.
    - `enumerate_coauthors`: same revwalk structure; for each commit, iterate `commit.message().unwrap_or("").lines()`, trim, call `strip_coauthor_prefix(trimmed)`, if Some pass the result to `parse_coauthor_value(rest.trim())`, if Some accumulate in `HashMap<(String, String), usize>`, convert to `Vec<CoAuthorEntry>`, sort desc.
    - Private `fn strip_coauthor_prefix(line: &str) -> Option<&str>`: const `PREFIX: &str = "co-authored-by:";` Check `line.len() >= PREFIX.len() && line[..PREFIX.len()].eq_ignore_ascii_case(PREFIX)` and slice off the prefix. Use `eq_ignore_ascii_case` not `to_lowercase` — avoids allocating per line.
    - Private `fn parse_coauthor_value(value: &str) -> Option<(String, String)>`: `let lt = value.rfind('<')?; let gt = value.rfind('>')?; if gt < lt { return None; }` then split into name (trimmed) and email (trimmed). If both empty, return None. Note: `rfind('<')`/`rfind('>')` are Unicode-safe because `<` and `>` are single-byte ASCII (Pitfall 9).
