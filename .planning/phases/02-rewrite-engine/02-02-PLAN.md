<objective>
Implement the rename-author half of the rewrite engine following strict TDD: write failing integration tests first (RED), then implement pub fn rewrite_author in src/git/rewrite.rs (GREEN). Closes requirements RENAME-03 (rewrite author AND committer when committer matches old author identity) and RENAME-04 (annotated tag objects recreated, not just ref pointer).

The engine walks all commits reachable from refs/heads/* and refs/tags/* in topological-reverse order, builds a HashMap of old-to-new commit OIDs, creates new commit objects for any commit whose identity matches OR whose parents were rewritten, then updates branch refs, annotated tag objects, lightweight tag refs, and detached HEAD in a single post-walk pass.

Purpose: rename is the first user-facing operation. Until this works, the tool does nothing. RENAME-04 (annotated tag recreation) is a Phase 2 constraint that must NOT be deferred to Phase 3.

Output: a complete rewrite_author library function with integration tests proving rename correctness across branches, merge commits, annotated tags, lightweight tags, detached HEAD, and byte-identity for non-target commits.

Do NOT push refs/remotes/* explicitly. Set sorting to Sort::TOPOLOGICAL bitor Sort::REVERSE — topological-reverse guarantees parents are visited before children so every parent OID is either already in oid_map or unchanged.
      Initialize: let mut oid_map: HashMap<Oid, Oid> = HashMap::new(); and let mut count: usize = 0;.
      For each iteration item (old_oid result from the revwalk), propagate the result via ? and call repo.find_commit(old_oid)?. Compute:
        identity_matches as the boolean: commit.author().name().unwrap_or("") equals old_name AND commit.author().email().unwrap_or("") equals old_email. The identity match is on AUTHOR ONLY — RENAME-03's committer-rewrite is a CONSEQUENCE of an author match, not an independent trigger.
