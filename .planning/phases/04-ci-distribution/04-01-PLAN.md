  <action>
    Create .github/workflows/ directory and write release.yml as follows.

    Workflow-level metadata:
      name: release
      on.push.tags: ["v[0-9]+.[0-9]+.[0-9]+"]  (constrained glob, NOT v*)
      permissions: contents: write

    Single job named build-release with strategy.fail-fast: false and matrix.include of exactly three entries:

    Entry 1 — Linux musl:
      build: linux-x86_64-musl
      os: ubuntu-latest
      target: x86_64-unknown-linux-musl
      binary: git-author-reformer
      asset_name: git-author-reformer-linux-x86_64
      rustflags: "-C target-feature=+crt-static"

    Entry 2 — macOS Apple Silicon:
      build: macos-aarch64
      os: macos-15
      target: aarch64-apple-darwin
      binary: git-author-reformer
      asset_name: git-author-reformer-macos-aarch64
      rustflags: ""
      (Note: uses macos-15 not macos-14; CLAUDE.md tech-stack table mentions macos-14 but that entry predates current runner availability; macos-15 is the current stable arm64 runner per RESEARCH.)

    Entry 3 — macOS Intel:
      build: macos-x86_64
      os: macos-15-intel
      target: x86_64-apple-darwin
      binary: git-author-reformer
      asset_name: git-author-reformer-macos-x86_64
      rustflags: ""
      (Note: uses macos-15-intel not macos-13; macos-13 was retired December 4, 2025. macos-15-intel replaces it. This is a native Intel runner — no cross-compilation involved.)

    Job steps in order:

    Step 1 — Checkout:
      name: Checkout
      uses: actions/checkout@v4

    Step 2 — Install musl tools (Linux only):
      name: Install build dependencies (Linux only)
      if: matrix.os == 'ubuntu-latest'
      run: |
        sudo apt-get update
        sudo apt-get install -y --no-install-recommends musl-tools cmake pkg-config
      (cmake and pkg-config added defensively for vendored-libgit2 C build path; musl-tools alone should suffice per jj/ripgrep evidence but these are cheap insurance per RESEARCH §Open Questions A2)

    Step 3 — Install Rust toolchain:
      name: Install Rust toolchain
      uses: dtolnay/rust-toolchain@stable
      with:
        targets: ${{ matrix.target }}

    Step 4 — Cache dependencies:
      name: Cache dependencies
      uses: Swatinem/rust-cache@v2
      with:
        key: ${{ matrix.target }}

    Step 5 — Verify Cargo.toml version matches tag:
      name: Check Cargo.toml version matches tag
      shell: bash
      run: |
        TAG="${{ github.ref_name }}"
        VERSION="${TAG#v}"
        if ! grep -q "^version = \"${VERSION}\"" Cargo.toml; then
          echo "Error: Cargo.toml version does not match tag ${TAG}" >&2
          exit 1
        fi

    Step 6 — Build release binary:
      name: Build release binary
      env:
        RUSTFLAGS: ${{ matrix.rustflags }}
      run: cargo build --release --target ${{ matrix.target }} --locked

    Step 7 — Prepare release asset (copy + generate checksum):
      name: Prepare release asset
      shell: bash
      run: |
        BIN="target/${{ matrix.target }}/release/${{ matrix.binary }}"
        cp "${BIN}" "${{ matrix.asset_name }}"
        shasum -a 256 "${{ matrix.asset_name }}" > "${{ matrix.asset_name }}.sha256"
      (shasum -a 256 works on both macOS and ubuntu-latest; sha256sum is Linux-only and must NOT be used)

    Step 8 — Upload to GitHub Release:
      name: Upload to GitHub Release
      uses: softprops/action-gh-release@v2
      with:
        files: |
          ${{ matrix.asset_name }}
          ${{ matrix.asset_name }}.sha256

    Step 9 — Verify Linux binary is static (Linux only):
      name: Verify Linux binary is static
      if: matrix.os == 'ubuntu-latest'
      shell: bash
      run: |
        ldd "${{ matrix.asset_name }}" 2>&1 | grep -q "not a dynamic executable" \
          || { echo "ERROR: Binary is not fully static!" >&2; ldd "${{ matrix.asset_name }}" >&2; exit 1; }

    YAML syntax requirements:
    - Indent with 2 spaces throughout
    - Multi-line run blocks use | (literal block scalar)
    - shell: bash on steps that use bash features (${TAG#v}, process substitution)
    - Do NOT add shell: bash to the musl apt-get step (plain sh is fine there)
    - Do NOT pin Swatinem/rust-cache or dtolnay/rust-toolchain to specific SHAs — use @stable and @v2 floating tags as specified by RESEARCH
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub tag push -> CI runner | Untrusted: any collaborator with push access can trigger a release build |
| CI runner -> GitHub Releases | CI runner writes artifacts; GITHUB_TOKEN controls write scope |
| GitHub Releases -> user machine | User downloads binary over HTTPS; checksum file downloaded from same Release |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | Binary transit (GitHub Releases -> user) | mitigate | SHA256 checksum file published alongside binary; install.sh verifies before chmod +x (implemented in 04-02) |
| T-04-02 | Tampering | GHA action supply chain | mitigate | Only approved actions per RESEARCH Package Legitimacy Audit: dtolnay/rust-toolchain, Swatinem/rust-cache@v2, softprops/action-gh-release@v2, actions/checkout@v4 — all verified maintained projects with years of adoption |
| T-04-03 | Spoofing | Release download source | accept | HTTPS + GitHub repo URL is sufficient for v1; no code-signing (macOS notarization deferred to v2) |
| T-04-04 | Tampering | Cargo.toml version != tag | mitigate | Explicit grep version-check step in workflow; job fails before building if mismatch detected |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new cargo, npm, or pip packages installed in this phase; only GitHub Actions (audited in RESEARCH) and apt packages (musl-tools, cmake, pkg-config — standard Ubuntu packages from official repos) |
</threat_model>
