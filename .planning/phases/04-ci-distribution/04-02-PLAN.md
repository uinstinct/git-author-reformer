Key install script constraints from RESEARCH.md:
- Shebang: #!/usr/bin/env sh  (POSIX sh, not bash)
- set -eu  (NOT set -euo pipefail — pipefail is bash-only, fails on macOS sh)
- shasum -a 256  (NOT sha256sum — sha256sum is Linux-only, fails on macOS)
- curl -fsSL  (fail on HTTP error, silent, follow redirects)
- Verify checksum BEFORE chmod +x (fail closed)
- Trap TMPDIR cleanup on EXIT — binary must run as a foreground child (NOT exec) so the trap fires after the binary finishes
- Binary names map: linux x86_64 -> linux-x86_64-musl, darwin arm64/aarch64 -> macos-aarch64, darwin x86_64 -> macos-x86_64
- REPO variable must be set to the actual GitHub owner/repo string
- VERSION env var override (default: "latest")
- One-shot runner: binary lives in TMPDIR_WORK and is deleted after the tool exits via trap
- detect_platform() takes two arguments (os, arch) — makes unit testing trivial without PATH manipulation

    Function: detect_platform() — takes two positional args: os ($1) and arch ($2).
    Returns the platform string via stdout. Exits non-zero with stderr message for unsupported combos.
      detect_platform() {
        _os="$1"
        _arch="$2"
        case "${_os}" in
          Linux)
            case "${_arch}" in
              x86_64) echo "linux-x86_64-musl" ;;
              *) printf 'Unsupported Linux arch: %s\n' "${_arch}" >&2; return 1 ;;
            esac ;;
          Darwin)
            case "${_arch}" in
              arm64|aarch64) echo "macos-aarch64" ;;
              x86_64)        echo "macos-x86_64" ;;
              *) printf 'Unsupported macOS arch: %s\n' "${_arch}" >&2; return 1 ;;
            esac ;;
          *)
            printf 'Unsupported OS: %s\n' "${_os}" >&2; return 1 ;;
        esac
      }

    Function: verify_checksum() — takes two positional args: binary_path ($1) and checksum_file ($2).
    Reads expected hash from checksum_file (first field of first line), computes actual hash of binary_path.
    Exits 1 with stderr message on mismatch.
      verify_checksum() {
        _bin="$1"
        _sha_file="$2"
        _expected="$(awk '{print $1; exit}' "${_sha_file}")"
        _actual="$(shasum -a 256 "${_bin}" | awk '{print $1}')"
        if [ "${_expected}" != "${_actual}" ]; then
          printf 'Checksum verification failed!\nExpected: %s\nActual:   %s\n' \
            "${_expected}" "${_actual}" >&2
          return 1
        fi
      }

    Main body guard — only run when script is executed directly, not sourced:
      # Detect if being sourced (for tests) vs executed
      # $0 contains the script name when executed; when sourced in sh, $0 is the parent shell name
      # Use a portable pattern: check if the script basename is in $0
      case "$0" in
        *install.sh) : ;;   # being executed — fall through to main body
        *)            return 0 ;;   # being sourced — stop here, functions are loaded
      esac

    Main body (only runs on direct execution):
      PLATFORM="$(detect_platform "$(uname -s)" "$(uname -m)")"

      if [ "${VERSION}" = "latest" ]; then
        VERSION="$(curl -fsSL "https://api.github.com/repos/${REPO}/releases/latest" \
          | grep '"tag_name"' | head -1 \
          | sed 's/.*"tag_name": *"\([^"]*\)".*/\1/')"
      fi

      BASE_URL="https://github.com/${REPO}/releases/download/${VERSION}"
      BINARY_NAME="git-author-reformer-${PLATFORM}"
      TMPDIR_WORK="$(mktemp -d)"
      trap 'rm -rf "${TMPDIR_WORK}"' EXIT

      curl -fsSL "${BASE_URL}/${BINARY_NAME}" -o "${TMPDIR_WORK}/git-author-reformer"
      curl -fsSL "${BASE_URL}/${BINARY_NAME}.sha256" -o "${TMPDIR_WORK}/git-author-reformer.sha256"

      verify_checksum "${TMPDIR_WORK}/git-author-reformer" "${TMPDIR_WORK}/git-author-reformer.sha256"

      chmod +x "${TMPDIR_WORK}/git-author-reformer"
      printf 'Checksum verified. Running git-author-reformer...\n' >&2
      "${TMPDIR_WORK}/git-author-reformer" "$@"
      # NB: foreground child, NOT exec. The shell process remains alive after the binary exits
      # so the EXIT trap fires and TMPDIR_WORK is removed. Using exec would replace the shell
      # process and the trap would never fire, leaking the temp directory.

    Critical correctness requirements:
    - verify_checksum called BEFORE chmod +x (guards against executing a corrupted binary)
    - Binary runs as foreground child, never exec (trap cleanup depends on shell surviving)
    - shasum -a 256 everywhere; NEVER sha256sum (Linux-only, fails on macOS)
    - set -eu; NEVER add pipefail (bash-only, fails on POSIX sh on macOS)
    - Use TMPDIR_WORK not TMPDIR (TMPDIR is a reserved env var; overwriting it breaks mktemp)
    - All user-facing messages to stderr (>&2) so they don't pollute stdout when piped
    - "$@" passes through all args to the binary

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Releases HTTPS -> user shell | User's shell downloads binary over HTTPS from GitHub CDN |
| .sha256 file -> local verification | SHA256 sidecar downloaded from same release; compared before execution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | Binary in transit (GitHub -> tmpdir) | mitigate | SHA256 checksum downloaded from same release, verified with shasum before chmod +x; mismatch exits 1 before any execution |
| T-04-06 | Tampering | Checksum file in transit | accept | Checksum and binary both from same GitHub release over HTTPS; attacker who can modify one can modify both; code-signing (macOS notarization) deferred to v2 |
| T-04-07 | Elevation of Privilege | chmod +x before checksum verify | mitigate | verify_checksum() called before chmod +x; verified by Test 7 which asserts no executable bit is set on mismatch |
| T-04-08 | Denial of Service | curl network failure | accept | curl -f flag fails on HTTP errors; set -eu causes script to exit; user sees clear curl error on stderr |
| T-04-09 | Tampering | TMPDIR_WORK leaks (exec pattern) | mitigate | Binary runs as foreground child (not exec); shell survives, EXIT trap fires and removes TMPDIR_WORK after binary exits |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package manager installs in this plan; install.sh uses only standard system tools (curl, shasum, awk, mktemp) available on all target platforms |
</threat_model>
