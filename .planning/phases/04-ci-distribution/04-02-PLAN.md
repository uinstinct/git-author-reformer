- One-shot runner: binary lives in TMPDIR and is deleted on exit; not installed to PATH

    Step 1 — Write tests/install_script_test.sh FIRST.

    The test harness is a plain POSIX sh script. Use a pass/fail counter pattern:

      PASS=0
      FAIL=0
      fail() { echo "FAIL: $1" >&2; FAIL=$((FAIL+1)); }
      pass() { echo "PASS: $1"; PASS=$((PASS+1)); }

    For platform-detection tests, extract the detection logic from install.sh into a block that can be sourced with a custom PATH. Pattern:
      - Write a fake uname script to /tmp/fake_uname_$$ that prints the desired OS on first call and arch on second call
      - Set PATH=/tmp:$PATH before sourcing the detection block
      - Source the detection-only portion of install.sh (or a separate detect.sh included by install.sh)

    Alternatively (simpler), the install.sh can export a detect_platform() function that takes uname_s and uname_m as arguments — this makes testing trivial without PATH manipulation:
      detect_platform() {
        local os="$1" arch="$2"
        case "${os}" in
          Linux) ... ;;
          Darwin) ... ;;
          *) echo "Unsupported OS: ${os}" >&2; return 1 ;;
        esac
      }
    Then tests call: detect_platform "Linux" "x86_64" and assert the return value.
    Use the function-argument approach — it avoids fragile PATH manipulation.

    For checksum tests (Tests 6-7), write a minimal shell fixture to TMPDIR, compute real hashes with shasum -a 256, then exercise the checksum verification block directly (source just that portion, or run a subshell with the download step mocked out).

    Test harness exit code: exit $FAIL at the end — non-zero if any test failed.

    Step 2 — Write install.sh to make all tests pass.

    install.sh structure:

    Line 1: #!/usr/bin/env sh
    Line 2: set -eu
    Blank line
    REPO="uinstinct/git-author-reformer"   (exact GitHub owner/repo)
    VERSION="${VERSION:-latest}"

    detect_platform() function taking no args, using real uname calls internally:
      OS="$(uname -s)"
      ARCH="$(uname -m)"
      case "${OS}" in
        Linux)
          case "${ARCH}" in
            x86_64) PLATFORM="linux-x86_64-musl" ;;
            *) echo "Unsupported Linux arch: ${ARCH}" >&2; exit 1 ;;
          esac
          ;;
        Darwin)
          case "${ARCH}" in
            arm64|aarch64) PLATFORM="macos-aarch64" ;;
            x86_64)        PLATFORM="macos-x86_64" ;;
            *) echo "Unsupported macOS arch: ${ARCH}" >&2; exit 1 ;;
          esac
          ;;
        *)
          echo "Unsupported OS: ${OS}" >&2; exit 1 ;;
      esac
      echo "${PLATFORM}"

    Version resolution block (when VERSION=latest, hit GitHub API):
      if [ "${VERSION}" = "latest" ]; then
        VERSION="$(curl -fsSL "https://api.github.com/repos/${REPO}/releases/latest" \
          | grep '"tag_name"' | head -1 \
          | sed 's/.*"tag_name": *"\([^"]*\)".*/\1/')"
      fi

    Download and verify block:
      PLATFORM="$(detect_platform)"
      BASE_URL="https://github.com/${REPO}/releases/download/${VERSION}"
      BINARY_NAME="git-author-reformer-${PLATFORM}"
      TMPDIR_WORK="$(mktemp -d)"
      trap 'rm -rf "${TMPDIR_WORK}"' EXIT

      curl -fsSL "${BASE_URL}/${BINARY_NAME}" -o "${TMPDIR_WORK}/git-author-reformer"
      curl -fsSL "${BASE_URL}/${BINARY_NAME}.sha256" -o "${TMPDIR_WORK}/git-author-reformer.sha256"

      EXPECTED="$(awk '{print $1}' "${TMPDIR_WORK}/git-author-reformer.sha256")"
      ACTUAL="$(shasum -a 256 "${TMPDIR_WORK}/git-author-reformer" | awk '{print $1}')"
      if [ "${EXPECTED}" != "${ACTUAL}" ]; then
        echo "Checksum verification failed!" >&2
        echo "Expected: ${EXPECTED}" >&2
        echo "Actual:   ${ACTUAL}" >&2
        exit 1
      fi

      chmod +x "${TMPDIR_WORK}/git-author-reformer"
      echo "Checksum verified. Running git-author-reformer..." >&2
      exec "${TMPDIR_WORK}/git-author-reformer" "$@"

    Critical details:
    - Use TMPDIR_WORK not TMPDIR (TMPDIR is a standard env var and overwriting it interferes with mktemp on some systems)
    - exec replaces the shell process with the binary; trap runs on EXIT so cleanup happens after exec (exec triggers EXIT in POSIX sh)
    - "$@" forwards any args passed to install.sh through to the binary
    - All diagnostic messages go to stderr (>&2); only intentional binary output goes to stdout
    - set -eu (POSIX sh); do NOT add pipefail
    - shasum -a 256 everywhere; zero occurrences of sha256sum
    - The trap cleanup removes TMPDIR_WORK including the binary after exec exits

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Releases HTTPS -> user shell | User's shell downloads binary over HTTPS from GitHub CDN |
| .sha256 file -> local verification | SHA256 sidecar downloaded from same release; compared before execution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | Binary in transit (GitHub -> tmpdir) | mitigate | SHA256 checksum downloaded from same release, verified with shasum before chmod +x; mismatch exits 1 before any execution |
| T-04-06 | Tampering | Checksum file in transit | accept | Checksum and binary both from same GitHub release over HTTPS; attacker who can modify one can modify both; code-signing (macOS notarization) deferred to v2 |
| T-04-07 | Elevation of Privilege | chmod +x before checksum verify | mitigate | Script explicitly runs checksum comparison before chmod +x; verified by test 6 |
| T-04-08 | Denial of Service | curl network failure | accept | curl -f flag fails on HTTP errors; set -eu causes script to exit; user sees clear error message |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package manager installs in this plan; install.sh uses only standard system tools (curl, shasum, awk, mktemp) available on target platforms |
</threat_model>
