  <behavior>
    `src/hook/write.rs` exposes two `pub(crate)` functions:

    - `atomic_write_executable(target: &Path, contents: &str) -> std::io::Result<()>`:
      1. Compute `tmp = target.with_extension(format!("tmp.{}", std::process::id()))`.
      2. Create `tmp`, write `contents.as_bytes()`, call `f.sync_all()` (durability), drop the file handle.
      3. On Unix: read tmp's permissions metadata, set mode to `0o755` via `PermissionsExt::set_mode`, write permissions back. On non-Unix: no-op (via `#[cfg(unix)]` / `#[cfg(not(unix))]`).
      4. `fs::rename(&tmp, target)` — atomic on same filesystem.

      ORDER IS LOAD-BEARING: set permissions BEFORE rename. RESEARCH §Pitfall 3 — `chmod` after `rename` leaves a window where a non-executable hook is in place and concurrent `git commit` fails.

    - `delete_hook(path: &Path) -> std::io::Result<()>`:
      - Wraps `fs::remove_file(path)`. No other behavior. Called when `remove_strip` empties the strip list (HOOK-10).

    Inline tests (`#[cfg(test)] mod tests`) — use `tempfile::TempDir` (already a dev-dependency, Cargo.toml:17):
    - `atomic_write_executable_creates_file_with_contents` — write `"hello"`, read back, assert equality.
    - `atomic_write_executable_overwrites_existing_file` — pre-populate target with old content; call writer; assert new content.
    - `atomic_write_executable_sets_mode_0755_on_unix` (`#[cfg(unix)]`) — `fs::metadata(path).unwrap().permissions().mode() & 0o777 == 0o755`.
    - `atomic_write_executable_cleans_up_tmp_file` — after success, the `.tmp.<pid>` sibling does NOT exist.
    - `atomic_write_executable_emits_lf_line_endings` — write `"line1\nline2\n"`; read raw bytes; assert no `\r\n` substring (defense-in-depth around Pitfall §4 — the Rust writer must never inject CR).
    - `delete_hook_removes_file` — create a file, call `delete_hook`, assert file no longer exists.
    - `delete_hook_returns_err_on_missing_file` — calling on a path that doesn't exist returns `Err` (matches `fs::remove_file` semantics — Plan 04 glue handles this).

    Replace the three stub bodies in `src/hook/mod.rs` with real implementations following the state-machine described in RESEARCH §Architecture Patterns "System Architecture Diagram":

    `read_strip_list(repo) -> Result<HookState, AppError>`:
    1. `let path = path::commit_msg_hook_path(repo);`
    2. If `!path.exists()`, return `Ok(HookState::Absent)`.
    3. Read the file via `fs::read_to_string(&path)?`.
    4. If `parse::detect_markers(&contents).is_none()`, return `Ok(HookState::NotToolManaged(path))` (RESEARCH §Open Question 1).
    5. Otherwise return `Ok(HookState::Managed { emails: parse::extract_strip_list(&contents) })`.

    `install_strip(repo, email) -> Result<AddResult, AppError>`:
    1. Validate `email` is non-empty and `render::validate_email_for_embedding(email).is_ok()`; on failure return `Err(AppError::Io(io::Error::new(InvalidInput, "...")))` — pragmatic reuse of the existing `Io` variant for input validation, since adding a dedicated variant would violate Pitfall §6 "AppError enum bloat".
    2. Lowercase the input email via `.to_ascii_lowercase()` (RESEARCH §Open Question 2 — engine lowercases internally so Phase 6 callers don't have to).
    3. Determine current state via `read_strip_list(repo)?`:
       - `HookState::NotToolManaged(p)` → return `Err(AppError::HookExists(p))` (HOOK-06).
       - `HookState::Absent` → start from `emails = vec![]`.
       - `HookState::Managed { emails }` → start from these emails.
    4. If `emails.iter().any(|e| e.eq_ignore_ascii_case(&lowered_email))` → return `Ok(AddResult::AlreadyStripped)` WITHOUT writing anything (HOOK-05).
    5. Otherwise `emails.push(lowered_email)`; call `write::atomic_write_executable(&path, &render::render_hook(&emails))?`; return `Ok(AddResult::Installed { count: emails.len() })`.

    `remove_strip(repo, email) -> Result<RemoveResult, AppError>`:
    1. `let path = path::commit_msg_hook_path(repo);`
    2. Read state via `read_strip_list(repo)?`:
       - `HookState::NotToolManaged(p)` → return `Err(AppError::HookExists(p))` (same gate — caller shouldn't be touching a non-tool hook).
       - `HookState::Absent` → return `Ok(RemoveResult::NotFound)`.
       - `HookState::Managed { mut emails }` → continue.
    3. Lowercase the input email. Filter `emails` to drop any entry matching case-insensitively. If no change, return `Ok(RemoveResult::NotFound)`.
    4. If resulting `emails` is empty: call `write::delete_hook(&path)?`; return `Ok(RemoveResult::HookDeleted)` (HOOK-10).
    5. Otherwise: call `write::atomic_write_executable(&path, &render::render_hook(&emails))?`; return `Ok(RemoveResult::Updated { remaining: emails.len() })`.

    HOOK-12 enforcement: NONE of the three functions calls anything in `crate::git::preflight`. Verified by grep in the acceptance criteria.
