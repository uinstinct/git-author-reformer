## Phase 5: CR-01 Shell Injection Fix (HOOK-08 Security)

**Status: FIXED and verified in current code.**

The critical finding from 05-REVIEW.md (CR-01) identified that `validate_email_for_embedding` did not reject the single-quote character (`'`), which would terminate the shell's single-quote context in the generated hook's awk invocation and allow arbitrary shell command injection.

Current code at `src/hook/render.rs:77`:
```rust
'\'' | '"' | '\\' | '\n' | '\r' => return Err(ch),
```

The single-quote (`'\''`) is now rejected. Regression test `validate_email_for_embedding_rejects_single_quote` (render.rs:290-301) asserts this with the exact error character.

Additionally, `install_strip` and `remove_strip` in mod.rs now validate all emails read back from disk (lines 72-78 and 107-113) — the WR-01 defense-in-depth gap from the review is also closed.
